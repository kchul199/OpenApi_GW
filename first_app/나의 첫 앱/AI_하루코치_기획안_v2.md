# AI 하루 코치 — 앱 기획안 v2.0

> 하루를 말하면, AI가 최적의 하루를 설계합니다.

---

## 1. 프로젝트 개요

### 앱 이름
**하루코치 (HaruCoach)**

### 한 줄 소개
자연어로 하루 할 일을 말하면 AI가 최적의 스케줄을 설계하고, 하루 끝에 맞춤 리뷰를 제공하는 스마트 일과 관리 앱

### 핵심 가치
- **간편함**: 복잡한 입력 없이 말하듯 적으면 끝
- **지능형 스케줄링**: AI가 사용자의 패턴과 에너지 리듬을 학습하여 최적 시간 배분
- **성장 추적**: 매일의 리뷰가 쌓여 나만의 생산성 인사이트로 발전
- **함께하는 동기부여**: 친구/동료와 달성률을 공유하며 서로 자극

### 타겟 사용자
업무와 개인 일정을 균형 있게 관리하고 싶은 **직장인 (25~40세)**
- 아침마다 오늘 뭘 해야 할지 정리가 안 되는 사람
- 할 일 앱은 써봤지만 꾸준히 쓰기 어려웠던 사람
- 퇴근 후 "오늘 뭐 했지?" 싶은 하루를 줄이고 싶은 사람
- 회의가 많고 야근이 잦은 한국 직장 환경에서 시간 관리가 절실한 사람

### 포지셔닝
> "Motion이나 Reclaim이 글로벌 캘린더 최적화 도구라면, 하루코치는 **한국 직장인의 하루 리듬에 맞춘 AI 코치**입니다."

기존 글로벌 AI 스케줄러들은 구글 캘린더 기반의 서양식 시간 관리에 초점을 맞추고 있습니다. 하루코치는 한국 직장인의 맥락(회의 중심 업무, 점심시간 활용, 퇴근 후 자기계발 니즈, 야근 패턴)을 깊이 이해하는 **맥락 인식형 AI 코치**로 차별화합니다.

### 플랫폼
- **1차**: iOS (Swift, SwiftUI)
- **2차**: Android (Kotlin, Jetpack Compose) — iOS 출시 후 3개월 내

## 2. 핵심 기능 정의

### 2.1 자연어 일과 입력

사용자가 자유롭게 텍스트를 입력하면 AI가 구조화된 할 일 목록으로 변환합니다.

**입력 예시**:
> "오늘 오전에 기획서 마무리하고, 점심 후에 팀 미팅 1시간, 저녁에 러닝 30분이랑 영어 공부 좀 하고 싶어"

**AI 변환 결과**:
| 시간 | 할 일 | 예상 소요 | 카테고리 |
|------|--------|-----------|----------|
| 09:00~11:00 | 기획서 마무리 | 2시간 | 업무 |
| 13:00~14:00 | 팀 미팅 | 1시간 | 업무 |
| 18:30~19:00 | 러닝 | 30분 | 건강 |
| 21:00~22:00 | 영어 공부 | 1시간 | 자기계발 |

**주요 로직**:
- 시간이 명시되지 않으면 AI가 사용자의 출근 시간, 에너지 패턴을 고려해 자동 배치
- 집중 업무는 오전, 가벼운 작업은 오후에 배치하는 기본 전략 적용
- 사용자가 드래그로 시간 조정 가능

#### ⚠️ AI 해석 확인 및 수정 UX

AI가 잘못 해석할 수 있으므로, 변환 후 반드시 **확인 단계**를 거칩니다.

**확인 플로우**:
1. AI 변환 결과를 카드 형태로 표시
2. 상단에 "이렇게 이해했어요, 맞나요?" 메시지
3. 각 카드를 탭하면 **인라인 편집 모드** 진입 (시간, 제목, 카테고리 수정)
4. "확정" 버튼으로 최종 승인 → 타임라인에 반영

**엣지 케이스 처리**:
| 상황 | AI 대응 |
|------|---------|
| 시간이 모호함 ("점심 후에") | 사용자의 평소 점심 시간 기반 추정 + "1시 이후로 잡았어요" 표시 |
| 소요 시간 미명시 ("영어 공부 좀") | 유사 활동 과거 데이터 기반 추정 (기본값 1시간) |
| 하루 일정이 과다 (16시간 초과) | "오늘 일정이 좀 빡빡해요. 우선순위를 정해볼까요?" 경고 |
| 반복 패턴 감지 ("매주 월요일 팀 미팅") | "이 일정을 매주 반복할까요?" 자동 제안 |
| 이전 수정 이력 반영 | "지난번에 러닝을 저녁 7시로 바꾸셨죠, 이번에도?" 학습 반영 |

**AI 학습 루프**:
- 사용자의 수정 내역을 서버에 저장
- 동일/유사 입력 시 이전 수정 사항을 반영한 결과 제공
- 주기적으로 프롬프트 개선에 수정 데이터 활용

### 2.2 스마트 스케줄 엔진

단순히 할 일을 나열하는 것이 아니라 **최적의 시간 배분**을 제안합니다.

**스케줄링 원칙**:
- 에너지 곡선 반영 — 개인별 집중력 높은 시간대에 중요 업무 배치
- 버퍼 타임 자동 삽입 — 일정 사이에 5~15분 여유 시간 확보
- 현실성 검증 — 하루에 넣을 수 있는 양을 초과하면 우선순위 조정 제안
- 반복 패턴 학습 — "매주 월요일 팀 미팅" 같은 패턴을 자동 인식
- 한국 직장 맥락 — 점심시간(12~1시) 보호, 야근 시 에너지 분배 조정

### 2.3 실시간 진행 추적

하루 동안 각 할 일의 진행 상태를 쉽게 업데이트합니다.

**기능 상세**:
- 타이머 연동 — 할 일 시작 시 자동 타이머, 포모도로 모드 지원
- 원탭 완료 — 스와이프 또는 탭으로 빠르게 완료 처리
- 알림 — 다음 일정 5분 전 부드러운 알림
- iOS 위젯 — 잠금 화면과 홈 화면에서 오늘의 진행률 확인
- Live Activity — 현재 진행 중인 할 일을 다이나믹 아일랜드에 표시

### 2.4 AI 하루 리뷰

하루가 끝나면 AI가 오늘 하루를 분석하고 맞춤 피드백을 제공합니다.

**리뷰 내용**:
- 달성률 요약 (예: "오늘 5개 중 4개 완료! 달성률 80%")
- 시간 분석 (카테고리별 투자 시간 파이 차트)
- AI 코멘트 (예: "오전에 집중력이 좋았네요. 기획서를 예상보다 30분 빨리 끝냈어요!")
- 내일을 위한 제안 (예: "오늘 못한 영어 공부, 내일 오전 8시에 넣어볼까요?")
- 주간/월간 트렌드 리포트
- **SNS 공유용 리뷰 카드** — 깔끔한 디자인의 달성률 카드를 인스타 스토리 등에 공유 가능

### 2.5 소셜 기능 (그룹 공유)

친구나 동료와 함께 하루를 공유하며 동기부여를 높입니다.

**기능 상세**:
- 그룹 생성 (최대 5명) — 소규모 그룹으로 밀도 있는 교류
- 달성률 보드 — 오늘의 달성률을 그룹 내 공유
- 응원 보내기 — 이모지 리액션으로 가볍게 응원
- 주간 MVP — 가장 꾸준히 달성한 멤버 하이라이트
- 프라이버시 — 달성률만 공유, 구체적인 할 일 내용은 비공개 기본값

**그룹 유형별 설계**:
| 유형 | 특징 | 적합한 규모 |
|------|------|-------------|
| 직장 동료 그룹 | 업무 달성률 중심, 비공개 그룹 | 3~5명 |
| 친구 그룹 | 자기계발/운동 목표 공유 | 2~4명 |
| 공개 챌린지 | 특정 테마(30일 운동 등) 참여 | 10~50명 |

**이탈 방지 설계**:
- 1일 1회 그룹 요약 알림 (과도한 알림 방지)
- 3일 연속 미활동 시 부드러운 넛지 메시지
- 그룹 최소 활동 기준: 주 3회 스케줄 생성
- 이탈 시 자동으로 그룹 규모 축소 (빈 자리 부각 방지)

## 3. 화면 구성 (IA)

```
하루코치
├── 온보딩
│   ├── 스플래시
│   ├── 간단 소개 (3장 스와이프)
│   ├── 생활 패턴 설문
│   │   ├── 출근/퇴근 시간
│   │   ├── 아침형/저녁형 선택
│   │   ├── 집중 잘 되는 시간대

│   ├── Apple 로그인

│   ├── 자연어 입력 영역
│   ├── AI 확인 카드 ("이렇게 이해했어요")
│   │   ├── 인라인 편집 모드
│   │   └── 확정 버튼
│   ├── 타임라인 뷰 (시간순 할 일 카드)
│   ├── 진행률 링 차트
│   └── 퀵 액션 (타이머 시작, 완료 처리)
│
├── 리뷰
│   ├── 오늘의 리뷰 카드
│   ├── SNS 공유 카드 생성
│   ├── 주간 리포트
│   ├── 월간 트렌드
│   └── AI 인사이트 상세
│
├── 소셜
│   ├── 그룹 목록
│   ├── 그룹 상세 (달성률 보드)
│   ├── 그룹 생성/초대
│   └── 응원 & 리액션
│
├── 설정
│   ├── 프로필
│   ├── 알림 설정
│   ├── 근무 시간 설정
│   ├── AI 성향 조절 (빡빡하게/여유롭게)
│   └── 데이터 관리
│
└── 위젯
    ├── 잠금 화면 — 다음 할 일
    ├── 소형 위젯 — 달성률 링
    └── 중형 위젯 — 오늘의 타임라인 요약
```

### 주요 사용자 플로우

**아침 플로우** (핵심 경험):
1. 앱 열기 → 빈 타임라인과 입력창이 보임
2. "오늘 할 일" 자연어로 입력
3. AI가 최적 스케줄 제안 → **"이렇게 이해했어요" 확인 카드** 표시
4. 필요시 카드 탭하여 인라인 수정
5. "확정" 탭 → 타임라인에 카드 배치

**저녁 플로우**:
1. 마지막 할 일 완료 처리
2. 알림: "오늘 하루 어땠나요? 리뷰 볼까요?"
3. AI 하루 리뷰 카드 확인
4. (선택) SNS 공유 카드 생성 → 인스타/카카오 공유
5. 내일 제안 확인 → 수락하면 내일 초안 자동 생성

## 4. 기술 아키텍처

### 클라이언트 (iOS)

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| UI 프레임워크 | SwiftUI | 선언형 UI, 위젯/라이브 액티비티 지원 |
| 아키텍처 | TCA (The Composable Architecture) | 상태 관리 명확, 테스트 용이 |
| 로컬 DB | SwiftData | Apple 네이티브, Core Data 대비 간결 |
| 네트워크 | Swift Concurrency + URLSession | 네이티브 비동기 처리 |
| 위젯 | WidgetKit | 잠금 화면, 홈 화면 위젯 |
| 알림 | UNUserNotificationCenter | 로컬 + 푸시 알림 |
| 온디바이스 ML | Core ML / Apple Intelligence | 간단한 패턴 매칭, 프라이버시 강화 |

### 서버 / 백엔드

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| 서버 | Firebase (초기) → 자체 서버 (확장 시) | 빠른 MVP, 점진적 마이그레이션 |
| 인증 | Firebase Auth (Apple 로그인) | 간편한 소셜 로그인 |
| DB | Firestore | 실시간 동기화, 그룹 기능에 적합 |
| AI API | Claude API (1차) / OpenAI GPT (대안) | 자연어 해석, 리뷰 생성 |
| 푸시 | APNs + Firebase Cloud Messaging | 안정적인 푸시 전달 |
| 분석 | Firebase Analytics + Mixpanel | 사용자 행동 분석 |

### AI 프롬프트 관리 전략

AI 기반 앱의 품질은 프롬프트 관리에 달려 있습니다. 하루코치는 최소 3가지 핵심 프롬프트를 사용하며, 이를 체계적으로 관리합니다.

**핵심 프롬프트**:
| 프롬프트 | 용도 | 호출 빈도 |
|----------|------|-----------|
| 스케줄 생성 | 자연어 → 구조화된 할 일 + 시간 배치 | 하루 1~3회 |
| 리뷰 생성 | 하루 데이터 → 분석 코멘트 + 제안 | 하루 1회 |
| 추천 생성 | 과거 패턴 → 내일 일정 제안 | 하루 1회 |

**관리 원칙**:
- **서버 사이드 프롬프트**: 프롬프트를 앱에 하드코딩하지 않고 Firebase Remote Config에서 관리 → 앱 업데이트 없이 AI 품질 개선 가능
- **버전 관리**: 각 프롬프트에 버전 번호 부여, A/B 테스트 시 버전별 성과 비교
- **사용자 컨텍스트 주입**: 프롬프트에 사용자의 출퇴근 시간, 에너지 패턴, 과거 수정 이력을 동적으로 삽입

### 하이브리드 AI 전략 (비용 최적화)

모든 요청을 AI API로 보내면 비용이 급증합니다. 로컬 처리와 API 호출을 적절히 분리합니다.

```
사용자 입력
    │
    ▼
[로컬 룰 엔진] ──── 단순 패턴 매칭 가능? ──── Yes ──→ 로컬 처리 (무료)
    │                                                    예: "매일 아침 운동 30분"
    No                                                       "어제와 같은 일정"
    │
    ▼
[AI API 호출] ──── 복잡한 자연어 해석 필요
                   예: "오전에 기획서 마무리하고 점심 후에..."
                       "오늘은 좀 여유롭게, 중요한 건 보고서만"
```

**예상 로컬 처리율**: 전체 요청의 30~40% (반복 패턴, 단순 명령)
**API 비용 절감 효과**: 월 기준 30~40% 절감

### 하루코치의 핵심 차별점

1. **"입력은 자유롭게, 관리는 AI가"**: 기존 앱들이 구조화된 입력을 요구하는 반면, 대화하듯 입력
2. **리뷰 + 코칭**: 단순 스케줄러가 아닌 하루 전체를 돌아보는 코치 역할
3. **한국 직장 맥락**: 점심시간, 회의 문화, 야근 패턴, 회식 등 한국 특유의 업무 환경 이해
4. **소셜 동기부여**: 혼자가 아닌 함께하는 경험으로 리텐션 확보
5. **가격 경쟁력**: 글로벌 앱 대비 50~70% 낮은 가격으로 한국 시장 공략
